\# AS-002 Event Quality Standard



Version: 1.0



Status: Active



Owner: Project Atlas



Created: 2026-07-11



\---



\# 1. Purpose



This Standard exists to protect the quality of Atlas knowledge.



Only events that create long-term investment knowledge may enter the Atlas Knowledge Graph.



Atlas records reality.



Atlas does not record noise.



\---



\# 2. Core Principle



Every event must satisfy all mandatory requirements.



If one requirement is not met,



the event SHALL NOT enter the Atlas database.



\---



\# 3. Mandatory Requirements



\## Rule 1



The event must have objectively occurred.



Examples:



✔ Government policy released



✔ Company announcement published



✔ Financial report disclosed



✘ Market rumor



✘ Personal prediction



\---



\## Rule 2



The event must be verifiable.



The original source must exist.



Examples:



✔ CSRC



✔ Shanghai Stock Exchange



✔ Shenzhen Stock Exchange



✔ Listed Company Announcement



\---



\## Rule 3



The event must influence capital markets.



The event should affect at least one of:



• Listed company



• Industry



• Market sentiment



• Economy



• Investors



Pure entertainment events are excluded.



\---



\## Rule 4



The event must have historical value.



Atlas records knowledge,



not temporary traffic.



The event should still be meaningful after:



• 1 month



• 1 year



• 10 years



\---



\## Rule 5



The event must be uniquely identifiable.



Each event shall have:



Event ID



Title



Date



Source



Category



Description



Related entities



\---



\# 4. Exclusion Rules



Atlas SHALL NOT record:



• Rumors



• Clickbait news



• Personal opinions



• Investment recommendations



• Target prices



• Unverified screenshots



• Anonymous social media posts



\---



\# 5. Event Review Process



Every new event shall pass:



1\. Source Verification



↓



2\. Quality Check



↓



3\. Importance Scoring



↓



4\. Relation Building



↓



5\. Database Entry



\---



\# 6. Compliance



Any event violating this Standard must be rejected.



\# AI Enforcement



Before inserting any event into Atlas, the AI must verify:



□ Event has objectively occurred



□ Source is verified



□ Event meets quality standard



□ Importance score calculated



□ Related entities identified



□ Event ID assigned



Only after all checks pass may the event enter the Atlas Knowledge Graph.
